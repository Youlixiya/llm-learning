从工程视角看 LLM 项目全流程

不只是“模型训练”和“Prompt 设计”，而是一条贯穿数据、模型、系统和评测的完整链路。

你可以这样使用本教程
  • Step 1 · 环境 + Tiny LM： 按 `README.md` 完成环境准备，运行 scripts/run_tiny_lm.sh 跑通第一个模型。
  • Step 2 · 理解与改造： 结合 docs/01_*.md 与 docs/02_*.md ，尝试改模型尺寸、数据和训练参数。
  • Step 3 · 上大模型 + LoRA： 参考 LoRA 章节与 src/finetune 代码，对开源模型做指令微调。
  • Step 4 · 接入你的业务： 利用 RAG、Agent 和 API 部署章节，把教程代码改造成与你领域相关的应用。
本教程特别强调的三件事
  • 工程导向： 更多讲“如何在有限算力下做事”“如何控制成本和风险”，少讲抽象理论。
  • 可复用性： `src/` 中的代码按模块组织，方便你拆出来用在自己的服务中。
  • 记录与迭代： 鼓励你在学习过程中记录实验、踩坑和配置，形成自己的 LLM 运维手册。
